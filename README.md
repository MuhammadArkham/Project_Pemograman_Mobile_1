2. StoryBoard Project
   ![foto](https://github.com/MuhammadArkham/Project_Pemograman_Mobile_1/blob/main/FOTO%20PROJECT/Storyboard.png?raw=true)
   Storyboard Aplikasi "FITME" (Kalkulator BMI)
Ini adalah alur cerita (storyboard) untuk aplikasi "FITME". Alur ini menjelaskan alur pengguna (user flow) utama, mulai dari membuka aplikasi, memasukkan data, hingga melihat hasil dan riwayat BMI.
Frame 1: Splash Screen
Visual: Layar penuh menampilkan logo aplikasi "FITME" (ikon hati) yang terpusat. Terdapat nama aplikasi "FITME" di bawah logo.
Deskripsi: Ini adalah layar pertama yang dilihat pengguna saat ikon aplikasi ditekan. Tujuannya adalah untuk branding dan memberikan waktu singkat bagi aplikasi untuk memuat sumber daya awal sebelum beralih ke Halaman Utama.
Frame 2: Halaman Utama
Visual: Menampilkan ilustrasi di bagian atas. Di bawahnya ada dua tombol pilihan aksi: "Hitung BMI Sekarang" dan "Hitung BMI". Terdapat bilah navigasi (navbar) di bagian bawah.
Deskripsi: Ini adalah dashboard atau layar utama aplikasi. Pengguna disajikan dengan opsi untuk langsung memulai alur utama, yaitu menghitung BMI.
Frame 3: Pilih Jenis Kelamin
Visual: Layar menampilkan judul "Pilih Jenis Kelamin". Terdapat dua opsi pilihan (radio button) dengan ikon: "Laki-Laki" dan "Perempuan". Di bagian bawah ada tombol "Simpan".
Deskripsi: Langkah pertama dalam alur "Hitung BMI" setelah pengguna memilih untuk menghitung. Pilihan ini wajib diisi karena rumus perhitungan BMI antara laki-laki dan perempuan mungkin berbeda.
Frame 4: Input Data
Visual: Sebuah formulir dengan panah kembali ("<") dan judul "Input Data". Terdapat tiga kolom input teks untuk "Tinggi Badan", "Berat Badan", dan "Tingkat Aktivitas". Di bagian bawah ada tombol "Hitung".
Deskripsi: Pengguna memasukkan data fisik mereka yang diperlukan untuk kalkulasi. Setelah semua kolom diisi, pengguna menekan tombol "Hitung" untuk memproses data.
Frame 5: Result Screen (Layar Hasil)
Visual: Menampilkan judul "Hasil BMI". Di bawahnya, angka hasil perhitungan ("22.5") ditampilkan besar. Terdapat juga status (misal: "Normal") dan grafik batang sederhana untuk memvisualisasikan rentang BMI. Di bawahnya ada tombol "Simpan".
Deskripsi: Layar ini menampilkan output atau hasil akhir dari kalkulasi BMI. Pengguna dapat melihat angka pasti, statusnya (apakah normal, kurus, atau gemuk), dan menyimpan hasil ini ke dalam riwayat mereka.
Frame 6: History Screen (Layar Riwayat)
Visual: Menampilkan judul "Riwayat Perhitungan". Terdapat grafik garis (line chart) yang menunjukkan tren hasil dari waktu ke waktu. Di bawah grafik, ada tabel data yang merinci tanggal (misal: "Hari ini", "Kemarin") dan angka BMI yang sesuai. Terdapat tombol "Simpan".
Deskripsi: Tampilan ini memungkinkan pengguna untuk melacak kemajuan dan melihat catatan perhitungan BMI mereka sebelumnya. Ini membantu pengguna memantau perubahan berat badan dan status kesehatan mereka dari waktu ke waktu.
